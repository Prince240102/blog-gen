# AGENTS.md — BlogForge

## Commands

```bash
# Full stack (requires API keys in .env)
docker compose up --build -d

# Local dev only (needs WordPress running separately)
uvicorn main:app --reload

# First-time setup after Docker starts
./setup-wp.sh

# Lint
ruff check app/

# Test
pytest
```

## Pipeline

**Research → SEO → Content → Humanize → Show to user → Publish**

- **SEO**: Makes actual improvements (keyword placement, readability, structure), not just ratings
- **Humanize**: Removes AI slop (em dashes, robotic language, etc.)
- **Publish**: Converts markdown to HTML, removes # tags for WordPress format

## Services

| Service | URL | Container |
|---------|-----|-----------|
| BlogForge API | http://localhost:8000 | blog-gen-api |
| WordPress | http://localhost:18888 | blog-gen-wp |
| WP Admin | http://localhost:18888/wp-admin | — |

## Required Env vars

```
OPENAI_API_KEY      # OpenAI GPT-4o
BRAVE_API_KEY       # Brave Search
WORDPRESS_URL       # http://wordpress (Docker) or http://localhost:8888 (local)
WORDPRESS_USERNAME  # WordPress user
WORDPRESS_APP_PASSWORD  # Generated by setup-wp.sh
JWT_SECRET          # Token secret
```

## Architecture

- **Entry**: `main.py` → `app/main.py` (FastAPI app)
- **Agents**: LangGraph pipeline in `app/agents/`
  - `research.py` → Brave Search
  - `seo.py` → Keyword analysis + content optimization
  - `content.py` → GPT-4o content generation
  - `humanize.py` → Remove AI slop (em dashes, robotic phrases)
  - `publisher.py` → WordPress REST API (converts markdown to HTML)
- **API**: `app/api/routes.py` + `app/api/pages.py`
- **Sessions**: In-memory (`app/services/session.py`) — lost on container restart

## Gotchas

- Sessions are in-memory — restart = new chat
- WordPress credentials in `.env` must match application password from `./setup-wp.sh`
- Brave Search called directly via `httpx` (PyPI package has broken metadata)
- Publishes as **draft** — log into WP Admin to publish publicly
- Publisher converts markdown to HTML automatically
